    Chapter 3: Code and Queries

    Code Samples

    In this text-based lecture, we will look at some examples of queries in different programming languages.

    A simple MQL query on the people collection

    Here is an example query in MQL.

    We will translate this query to four programming languages (Java, Python, .NET, and Javascript). This will highlight how the query can be translated to these languages and show how to run the query in these idiomatic drivers. In each example, we are connecting to a database named test and a collection named people.

    Scroll down to the language you're most familiar with and review how the query gets translated.

    Python code for the MQL query example

    Here is the previous MQL example translated to Python.

    In order to translate the query, we have added the necessary scaffolding to connect, to set the conditional, and query plus store the result of the query.

    # Requires the PyMongo package - https://pymongo.readthedocs.io/en/stable/installation.html
client = MongoClient('mongodb://localhost:27017/?readPreference=primary&appname=MongoDB%20Compass&ssl=false')
filter = {
    'age': {
        '$lt': 30
    }
}
test_db = client.test
people_coll = test_db.people
result = people_coll.find(
  filter = filter
)

    The PyMongo MongoClient class is used to connect to the local MongoDB deployment.

    client = MongoClient('mongodb://localhost:27017/?readPreference=primary&appname=MongoDB%20Compass&ssl=false')

    The test_db and people_coll variables are used to hold the intermediate database and collection objects returned by the driver.

    test_db = client.test
people_coll = test_db.people

    The filter variable stores the actual condition we want to query on. It is identical to the MQL query.

    filter = {
    'age': {
        '$lt': 30
    }
}

    The result variable stores the result of the execution of the query against the MongoDB database.

    result = people_coll.find(
  filter = filter
)

    Congratulations, we've successfully translated the query, learnt about the necessary scaffolding required to connect to the database, how to construct the conditional elements of the query, and how to store the results from the query in Python.

    Java code for the MQL query example

    Here is the same MQL example translated to Java. We have excluded the Java library packages to improve the readability of this example.

    // Requires the MongoDB Java Driver - https://mongodb.github.io/mongo-java-driver
Bson filter = lt("age", 30L);
MongoClient mongoClient = new MongoClient(
    new MongoClientURI(
       "mongodb://localhost:27017/?readPreference=primary&appname=MongoDB%20Compass&ssl=false"
    )
);
MongoDatabase database = mongoClient.getDatabase("test");
MongoCollection<Document> collection = database.getCollection("people");
FindIterable<Document> result = collection.find(filter);

    The filter variable stores the actual condition we want to query on. It is functionality equivalent to the MQL criteria.

    Bson filter = lt("age", 30L);

    The MongoDB Java Driver MongoClient class is used to connect to the local MongoDB deployment.

    MongoClient mongoClient = new MongoClient(
    new MongoClientURI(
       "mongodb://localhost:27017/?readPreference=primary&appname=MongoDB%20Compass&ssl=false"
    )
);

    The database and collection variables are used to hold the intermediate database and collection objects returned by the driver.

    The result variable stores the result of the execution of the query against the local MongoDB database.

    MongoDatabase database = mongoClient.getDatabase("test");
MongoCollection<Document> collection = database.getCollection("people");
FindIterable<Document> result = collection.find(filter);

    Well done. We have just converted the query from MQL to Java. In a similar fashion to Python, we've seen how to connect to the database, construct the query, then execute the query, and finally store the result in Java.

    .NET code for the MQL query example

    Here is the same MQL example translated to .NET. We have excluded the .NET library packages to improve the readability of this example.

    // Requires the MongoDB .NET Driver - https://mongodb.github.io/mongo-csharp-driver/2.10/getting_started/installation/
var client = new MongoClient("mongodb://localhost:27017");
var database = client.GetDatabase("test");
var collection = database.GetCollection<BsonDocument>("people");
var filter = Builders<BsonDocument>.Filter.Lt("age", 30);
var document = collection.Find(filter).First();

    The .NET CSharp MongoClient class is used to connect to the local MongoDB deployment and the client variable holds the connection.

    var client = new MongoClient("mongodb://localhost:27017");

    The filter variable stores the actual condition we want to query on. It is functionality equivalent to the MQL criteria.

    var filter = Builders<BsonDocument>.Filter.Lt("age", 30);

    The database and collection variables are used to hold the intermediate database and collection objects returned by the driver.

    The result variable stores the result of the execution of the query against the local MongoDB database.

    MongoDatabase database = mongoClient.getDatabase("test");
MongoCollection<Document> collection = database.getCollection("people");
FindIterable<Document> result = collection.find(filter);

    In a similar fashion to Python and to Java, we've now seen how to translate the query and perform the necessary operations to query then store the results in .NET C#.

    Javascript code for the MQL query example

    Here is the same MQL example translated to Node. We have excluded the Node library packages to improve the readability of this example.

    /*
 * Requires the MongoDB Node.js Driver
 * https://mongodb.github.io/node-mongodb-native
 */

const filter = {
  'age': {
    '$lt': 30
  }
};

MongoClient.connect(
  'mongodb://localhost:27017/?readPreference=primary&appname=MongoDB%20Compass&ssl=false',
  { useNewUrlParser: true, useUnifiedTopology: true },
  function(connectErr, client) {
    assert.equal(null, connectErr);
    const coll = client.db('test').collection('people');
    coll.find(filter, (cmdErr, result) => {
      assert.equal(null, cmdErr);
    });
    client.close();
  });

    The filter variable stores the actual condition we want to query on. It is identical to the MQL query.

    const filter = {
      'age': {
        '$lt': 30
      }
    };

    The Javascript NodeJS MongoClient class is used to connect to the local MongoDB deployment.

    MongoClient.connect(
  'mongodb://localhost:27017/?readPreference=primary&appname=MongoDB%20Compass&ssl=false',
  { useNewUrlParser: true, useUnifiedTopology: true }

    The filter variable is used to populate the query which is then run against the MongoDB database.

    coll.find(filter, (cmdErr, result) => {
      assert.equal(null, cmdErr);
    });

    Congratulations, you've now seen how to translate the query in four different programming languages.

    Regardless if your favorite programming language is Python, Java, .NET, Javascript, or some other language, MongoDB's idiomatic drivers make it easy and natural to query the database.

    CRUD in SQL and in MongoDB

    To learn more about how the MongoDB compares with the structured query language (SQL) for Create/Read/Update/Delete (CRUD) functionality, have a look at the following resource:

    • SQL to MongoDB Mapping Chart

    Here is a reference table to help translate between SQL functions into the equivalent in terms of MongoDB CRUD functions.

    SQL
    CRUD - Functions
    MongoDB
    CRUD - functions
    INSERT INTO insertOne() / insertMany()
    SELECT findOne() / find() / distinct()
    UPDATE updateOne() / updateMany() / replaceOne()
    DELETE deleteOne() / deleteMany()
    CREATE INDEX createIndex()
    DROP INDEX dropIndex() / dropIndexes()
    DROP TABLE drop()
    ALTER updateMany() with $set or with $unset for adding or removing fields within a collection

    Learning more about MongoDB Drivers

    The MongoDB Drivers documentation page has a full list of all the drivers in the MongoDB ecosystem. They all offer CRUD functionality: Create, Read, Update, and Delete operations on documents.

    MongoDB's idiomatic drivers minimize onboarding time for new developers and simplify application development.
